# Commits em Lotes

Dadas as modificações identificadas no projeto (git status):

1. Analise todas as alterações.
2. Liste todas as alterações por escopo (front, back, infra, arquitetura etc).
3. Informe ao usuário os lotes de arquivos junto com as mensagens de commit para cada lote, seguindo as convenções de commit semântico.
    - Cada lote informado deve conter a mensagem de commit (em português) seguida dos arquivos.
    - Cada lote deve depender do lote anterior.
4. Somente APÓS a confirmação do usuários é que de fatos, os commits devem ser feitos, de acordo com o planejado no passo anterior.
5. Após os commites serem executados, apresente um relatório direto com os todos os commits realizados.
6. Pergunte se o usuário quer fazer a ação de push ou se você pode o fazer por ele.
